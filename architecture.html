Multi-tier Architecture

Multi-tier architecture is a client-server software architecture where the presentation, application processing, and data management functions are physically separated into distinct layers[cite: 120, 122, 124].

This separation allows developers to modify or upgrade one tier without affecting the others, which improves the scalability, security, and maintenance of complex web applications[cite: 125].

The Three Tiers

  • Presentation Tier (UI/Br): The top level that users interact with directly[cite: 126].
  • Logic/App Tier (Server): The middle layer that processes commands and makes logical decisions[cite: 129].
  • Data Tier (Database): Where information is stored and retrieved[cite: 130].
